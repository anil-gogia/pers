for Optum Developer Portal support.

Briefly, here’s how the portal works:

There are two kinds of APIs, public APIs and private APIs.
In the case of public APIs, you will be able to call them with no further approval needed
In the case of private APIs, you will NOT be able to call them until you have been given permission by the API owner. 
To get this permission in production, the API Owner will need to submit an incident ticket to the “API Developer Portal” group on service now.
You are the member of a consumer organization, and access to private APIs is granted on an organization level. 

To Consume an API:
1.            Log into [ REPLACE WITH THE PORTAL URL THEY NEED TO LOG INTO ]
2.            Navigate to the dashboard (View API Manager Dashboard link on the landing page)
3.            In the Nav on the left, select the Applications tab
4.            Click “Add Application”
5.            Name it (for your own organizational purposes), and for the platform select “Hybrid”
6.            In the API Management tab, select the APIs you want to consume (only public APIs, and those private APIs which you have access to will be visible) 
7.            In the Auth tab leave the callback url blank, put “default” for the scope and select “Confidential” for the type
8.            Save
9.            Wait for the application to be approved (usually takes a couple of minutes)
10.          When the application is approved (will show Active, under the state) select “edit” under the gear menu, and your OAuth client & secret will be visible under the Auth tab
